향후 추가 학습

이 장은 기본기를 마친 분을 위한 다음 주제 안내입니다. 각 주제에 대해 "왜 필요한가"와 "어디서부터 시작하면 되는가"만 짧게 적습니다. 실제 구현은 이 가이드의 범위를 넘어서며, 공식 문서와 외부 자료로 이어집니다.

예상 학습 시간 · 10분 레퍼런스

A. Streamable HTTP 방식의 MCP 서버

왜 필요한가

지금까지 만든 서버는 stdio transport로, 같은 PC의 호스트(Claude Desktop 등)만 쓸 수 있습니다. 다음과 같은 경우엔 네트워크 기반 배포가 필요합니다.

  • 여러 팀원이 하나의 서버를 공유하고 싶다.
  • 사내 서버나 클라우드에 올려 두고 웹 기반 AI 도구에서 붙이고 싶다.
  • 서버 리소스(DB 커넥션, GPU 등)를 한곳에 집중시키고 싶다.

기본 아이디어

FastMCP는 mcp.run()의 옵션만 바꾸면 HTTP 기반으로 기동할 수 있습니다. 아래 코드만 기억하세요.

python
if __name__ == "__main__":
    # 기본 stdio 대신 Streamable HTTP 로 전환
    mcp.run(transport="streamable-http", port=8000)

또는 커맨드라인에서 실행:

bash
fastmcp run railway_server.py --transport streamable-http --port 8000

그 다음 호스트 측(Claude Desktop 설정 등)에는 실행 경로 대신 URL(http://your-host:8000/mcp)을 등록합니다.

같이 고민해야 하는 것들

  • 인증: 누구나 URL을 알면 접속되는 상태는 위험합니다. Bearer 토큰, OAuth 2.0, 사내 SSO 연동을 고려하세요.
  • 네트워크: HTTPS 적용, 리버스 프록시(Nginx), 방화벽 정책.
  • 배포: Docker 컨테이너화, systemd로 데몬화, 관측(로그/메트릭) 구성.
  • 세션·상태: 사용자별 상태가 있는 Tool은 세션 관리가 필요합니다.

출발점: FastMCP 공식 문서의 "Deployment" 섹션과 MCP 공식 문서의 "Transports" 장.

B. 기존 타언어 코드(MATLAB/C/C++/R) 래핑

현업에 이미 존재하는 알고리즘 자산(MATLAB, C/C++, R 등)을 Python으로 감싸서 MCP Tool로 노출하는 전략입니다. 공통 원칙은 같습니다.

핵심은 가운데 "Python 래퍼 함수"를 순수 함수로 만드는 것입니다. 그 다음 @mcp.tool을 붙이기만 하면 됩니다.

B-1. MATLAB 코드 래핑

두 가지 접근이 있습니다.

방식장점주의점
MATLAB Engine API for Python
matlab.engine 패키지		 MATLAB 워크스페이스를 그대로 활용. 함수·변수 모두 호출 가능. MATLAB 설치 필요, 라이선스 필요. 초기 엔진 기동이 느림(수 초).
MATLAB Compiler SDK로 사전 컴파일 라이선스 없이 배포 가능(MATLAB Runtime만 있으면 됨). 컴파일 단계가 추가. 디버깅 난이도 상승.

출발점 문서: MathWorks 공식 "MATLAB Engine API for Python".

B-2. C / C++ 코드 래핑

도구언제 쓰나
ctypes (표준 라이브러리)이미 .dll / .so가 있을 때. 별도 빌드 없이 사용 가능.
cffi헤더 파일 기반으로 바인딩을 선언적으로 생성. ctypes보다 안전함.
pybind11C++ 클래스·템플릿까지 노출하고 싶을 때. 가장 고성능·유연함.

C/C++로 짠 "계산 커널"을 pybind11로 감싸면, 거의 네이티브 속도로 MCP Tool 안에서 호출할 수 있습니다.

B-3. R 코드 래핑

  • rpy2 — Python에서 R 인터프리터를 임베드. R 함수·데이터프레임을 주고받을 수 있음.
  • subprocess로 Rscript 호출 — 단순하지만 강력. R 스크립트를 외부 프로세스로 실행하고 JSON으로 결과 회수.

R 환경이 복잡하지 않다면 subprocess가 의외로 실용적입니다.

공통 설계 패턴

어떤 언어든, 07장에서 다룬 설계 지침이 그대로 적용됩니다.

python · pattern.py (개념 예시)
from pydantic import BaseModel
from fastmcp import FastMCP

class SimResult(BaseModel):
    status: str
    value: float

def _call_legacy_matlab(x: float) -> float:
    # matlab.engine 또는 compiled binary 등 (생략)
    ...

mcp = FastMCP("legacy-bridge")

@mcp.tool
def run_simulation(x: float) -> SimResult:
    """기존 MATLAB 시뮬레이션을 실행하여 결과를 반환합니다.

    Args:
        x: 입력 파라미터.
    """
    value = _call_legacy_matlab(x)
    return SimResult(status="ok", value=value)

핵심: 타언어 호출은 전부 _로 시작하는 내부 함수에 감춰 두고, Tool 함수는 평범한 Python 함수처럼 보이게 유지합니다. 나중에 백엔드 구현(MATLAB → C++ → …)을 바꿔도 Tool 시그니처는 그대로입니다.
