개발 환경 설정

코드를 실행하려면 먼저 도구를 준비해야 합니다. Python 설치부터 MCP 서버 라이브러리 설치까지, Windows 기준으로 순서대로 따라와 주세요.

예상 학습 시간 · 20~30분 관리자 권한 필요할 수 있음

준비할 도구 체크리스트

이 장을 끝내면 아래 항목이 모두 갖춰집니다.

  • Python 3.11 이상
  • 코드 에디터(Visual Studio Code)
  • 가상환경(venv)
  • MCP 서버 라이브러리(fastmcp)
  • (선택) 호스트 프로그램 Claude Desktop

Step 1. Python 설치

파이썬 공식 사이트에서 설치 파일을 받습니다.

  1. python.org/downloads 접속 → "Download Python 3.13" 등 최신 3.11+ 버전 선택.
  2. 내려받은 설치 파일을 실행합니다.
  3. 매우 중요: 설치 첫 화면에서 "Add python.exe to PATH" 체크박스를 반드시 켜세요.
  4. "Install Now" 클릭 → 완료 대기.
가장 자주 빠뜨리는 실수

"Add python.exe to PATH"를 체크하지 않으면, 명령 프롬프트에서 python을 입력했을 때 "명령을 찾을 수 없습니다" 오류가 납니다. 이미 설치하셨다면 제어판에서 파이썬을 제거하고 다시 설치하거나, 하단 트러블슈팅을 참고하세요.

설치 확인

Windows 검색창에 "cmd"를 입력해 명령 프롬프트를 엽니다. 그리고 아래 명령을 한 줄씩 입력해 보세요.

bash
python --version
pip --version

아래처럼 나오면 성공입니다.

출력 예시
Python 3.13.2
pip 24.3.1 from C:\Users\...\site-packages\pip (python 3.13)
macOS 사용자

macOS는 python 대신 python3, pip 대신 pip3로 부르는 경우가 많습니다. 설치는 Homebrew로 brew install python@3.13 한 줄이면 끝납니다.

Step 2. 코드 에디터 설치

코드는 메모장에서도 작성 가능하지만, Visual Studio Code(VS Code)를 쓰면 자동완성·오류 표시가 있어 훨씬 편합니다.

  1. code.visualstudio.com에서 Windows용 설치 파일 다운로드 → 설치.
  2. VS Code 실행 후 왼쪽 "확장(Extensions)" 아이콘을 눌러 "Python" 확장(Microsoft 제공)을 검색해 설치.
Claude Desktop도 함께

나중에 우리가 만든 MCP 서버를 Claude에서 실제로 써보려면 Claude Desktop을 미리 설치해 두면 좋습니다. 설치만 해두고, 설정은 04장에서 함께 진행합니다.

Step 3. 프로젝트 폴더 만들기

MCP 서버 하나를 담을 폴더를 새로 만듭니다. 경로는 여러분이 편한 곳이면 됩니다. 여기서는 예시로 C:\mcp-first를 사용하겠습니다.

bash (cmd)
mkdir C:\mcp-first
cd C:\mcp-first

이후 모든 명령은 이 폴더 안에서 실행한다고 가정합니다.

Step 4. 가상환경(venv) 만들기

왜 필요한가? 프로젝트마다 쓰는 라이브러리 버전이 다를 수 있습니다. 가상환경은 "프로젝트 전용 작은 방"을 만들어, 한 프로젝트의 설치가 다른 프로젝트에 영향을 주지 않게 격리해 줍니다. MCP 서버를 여러 개 만들 때 이 습관이 여러분을 구해 줍니다.

bash (cmd, Windows)
python -m venv .venv
.venv\Scripts\activate

두 번째 명령 실행 후 프롬프트 맨 앞에 (.venv)가 붙으면 성공입니다.

출력 예시
(.venv) C:\mcp-first>
PowerShell 사용자 / macOS 사용자

PowerShell: .venv\Scripts\Activate.ps1. 만약 "스크립트 실행이 금지됨" 오류가 나면 PowerShell을 관리자 권한으로 열어 Set-ExecutionPolicy -Scope CurrentUser RemoteSigned를 한 번 실행한 뒤 다시 시도하세요.

macOS/Linux: source .venv/bin/activate

Step 5. FastMCP 설치

이제 본격적으로 MCP 서버 개발에 쓸 라이브러리 fastmcp를 설치합니다. 가상환경이 활성화된 상태에서 실행하세요.

bash
pip install fastmcp

수십 개의 메시지가 지나간 뒤 "Successfully installed ..."가 나오면 완료입니다. 설치 확인:

bash
pip show fastmcp

버전 정보가 출력되면 성공입니다.

참고: fastmcp 와 공식 mcp 패키지의 관계

MCP Python 생태계에는 크게 두 패키지가 있습니다.

  • fastmcp(이 가이드 기준) — 가장 먼저 등장한 고수준 프레임워크. 현재는 독립 프로젝트로 활발히 발전 중이며, OAuth·배포 도구 등 고급 기능까지 포함합니다.
  • mcp(공식 Python SDK, pip install "mcp[cli]") — 공식 modelcontextprotocol/python-sdk 저장소가 제공. 내부에 mcp.server.fastmcp.FastMCP라는 거의 동일한 고수준 API가 포함되어 있습니다.

두 패키지 모두 @mcp.tool로 도구를 등록하는 기본 API는 같습니다. 이 가이드는 설치 경로가 더 간결하고 문서가 친절한 fastmcp를 기준으로 설명합니다.

Step 6. 동작 확인: 가장 작은 서버

설치가 잘 되었는지 확인하기 위해 "인사말을 돌려주는 서버"를 5줄로 만들어 봅시다. 완벽히 이해할 필요는 없습니다. 지금은 실행이 되는지만 확인해 주세요.

VS Code에서 C:\mcp-first 폴더를 열고(File → Open Folder), hello.py 파일을 새로 만들어 아래 코드를 입력합니다.

python · hello.py
from fastmcp import FastMCP

mcp = FastMCP("hello-server")

@mcp.tool
def hello(name: str) -> str:
    """이름을 받아 인사말을 반환합니다."""
    return f"안녕하세요, {name}님!"

if __name__ == "__main__":
    mcp.run()

명령 프롬프트에서 실행:

bash
python hello.py

아무 오류 메시지 없이 화면이 가만히 멈춰 있다면 성공입니다. 이 상태가 "서버가 요청을 기다리고 있는" 상태입니다. Ctrl + C로 종료하세요.

정상 동작인데 왜 아무것도 안 보일까?

MCP 서버는 기본적으로 표준입출력(stdio)으로 클라이언트와 통신합니다. 클라이언트(Claude Desktop 등)가 붙기 전에는 조용히 기다리는 게 정상이에요. 실제로 동작을 보려면 06장의 MCP Inspector가 필요합니다.

자주 발생하는 오류와 대처

"'python'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램... 이 아닙니다."

PATH 문제입니다. 설치 시 "Add python.exe to PATH"를 체크하지 않은 경우. Python을 제거한 뒤 다시 설치하면서 체크박스를 켜는 방법이 가장 빠릅니다. 또는 설치 위치(%LOCALAPPDATA%\Programs\Python\Python313)를 시스템 환경 변수 Path에 직접 추가해도 됩니다.

"pip install fastmcp" 에서 SSL 또는 프록시 오류

사내 프록시 뒤에서 작업 중이라면, 사내 IT팀에서 안내한 HTTP_PROXY·HTTPS_PROXY 환경 변수를 설정하거나, pip install --proxy http://user:pass@proxy.company:8080 fastmcp 형태로 프록시를 명시하세요.

".venv\Scripts\activate" 가 인식되지 않음 (PowerShell)

PowerShell은 기본적으로 스크립트 실행이 제한되어 있습니다. Set-ExecutionPolicy -Scope CurrentUser RemoteSigned을 한 번 실행한 뒤 새 창을 여세요. 또는 cmd를 사용하면 이 문제는 없습니다.

"ModuleNotFoundError: No module named 'fastmcp'"

가상환경을 활성화하지 않은 채 실행하고 있기 때문입니다. 명령 프롬프트 맨 앞에 (.venv)가 붙어 있는지 확인하세요. 안 붙어 있다면 .venv\Scripts\activate부터.

중간 점검

여기까지 따라오셨다면 "Python이 설치되었고, fastmcp가 설치되었고, 아주 작은 서버가 실행되었다"는 뜻입니다. 훌륭합니다! 이제 다음 장에서 MCP 서버 코드에 꼭 필요한 Python 문법만 빠르게 짚고 갑니다.
