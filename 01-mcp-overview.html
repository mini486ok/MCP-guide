MCP 개요와 필요성

MCP가 무엇이고, 왜 AI 시대에 새로운 "표준"이 필요한지부터 먼저 짚습니다. 비개발자도 알 수 있도록 생활 속 비유를 씁니다.

예상 학습 시간 · 10분 사전 지식 없음

먼저 한 가지 비유부터

유능한 요리사(LLM) 한 명을 고용했다고 상상해 봅시다. 이 요리사는 세상의 거의 모든 레시피를 외우고 있고, 한국어·영어도 자유롭게 구사합니다. 그런데 중요한 문제가 하나 있습니다.

요리사에게 없는 것

요리사에게는 주방 도구가 없습니다. 냉장고를 열어 재료를 확인할 수도 없고, 오븐을 켤 수도 없고, 주문장을 볼 수도 없습니다. 말(지식)만 잘할 뿐, 실제 행동을 할 수 없는 겁니다.

LLM(Claude, ChatGPT 같은 AI)도 똑같습니다. 지식은 풍부하지만 다음과 같은 일은 직접 하지 못합니다.

  • 사내 데이터베이스에서 오늘의 열차 운행 상태를 조회하기
  • Excel 파일을 열어 특정 셀의 값을 읽기
  • 내가 MATLAB으로 짜 둔 시뮬레이션을 실행하기
  • 회사 시스템에 이메일을 보내기

이 일들을 해내려면 요리사 손에 도구를 쥐어 줘야 합니다. 그런데 도구마다 모양·규격이 제각각이면 요리사가 도구를 쓰는 법을 매번 새로 배워야겠죠. 이 문제를 해결해 주는 표준이 바로 MCP입니다.

MCP란 무엇인가요?

MCP(Model Context Protocol)는 AI 모델(LLM)이 외부 도구·데이터에 접근할 수 있도록 2024년 후반 Anthropic이 공개한 개방형 표준 프로토콜입니다. 2026년 현재 Claude, Cursor, VS Code 등 주요 AI 도구들이 공식적으로 지원합니다.

쉬운 한 줄 요약:

한 줄 정의

MCP는 "AI와 외부 도구를 이어주는 공용 어댑터"입니다. USB-C 포트처럼, 한 번만 규격에 맞게 만들어 두면 어떤 AI 앱에서도 그대로 꽂아 쓸 수 있습니다.

왜 지금 MCP가 필요한가요?

MCP가 등장하기 전에는, 예를 들어 "Claude에게 사내 DB를 연결" 하려면 Claude 전용 커넥터를, "ChatGPT에게 연결" 하려면 ChatGPT 전용 플러그인을 각각 따로 만들어야 했습니다. 같은 DB를 연결하는데도 말이죠.

MCP 이전 (N × M 통합 지옥)
AI 앱 3개  ×  도구 4개
  = 12개의 커넥터를 각각 개발

Claude  →  DB
Claude  →  이메일
Claude  →  MATLAB
Claude  →  파일
ChatGPT →  DB
ChatGPT →  이메일
ChatGPT →  MATLAB
... (계속)
MCP 이후 (표준 하나)
도구 4개만 MCP 규격으로 만들면
   어떤 AI 앱에서도 그대로 사용

[MCP 서버: DB]
[MCP 서버: 이메일]
[MCP 서버: MATLAB]
[MCP 서버: 파일]
        ↑ ↑ ↑ ↑
    Claude / Cursor / VS Code
    / 다른 AI 앱 모두 공통 접속

즉, "내가 만든 도구 하나를 여러 AI 앱이 공유"할 수 있게 되는 것이 MCP의 핵심 가치입니다.

MCP의 세 가지 등장인물

MCP를 이해하려면 세 역할을 구분해야 합니다. 이 가이드에서 우리는 서버를 직접 만들 겁니다.

호스트 안에는 MCP 클라이언트가 여러 개 있고, 각 클라이언트가 서버 하나씩과 통신합니다.
  • 호스트(Host) — 사람이 쓰는 AI 앱. Claude Desktop, Cursor IDE, VS Code 확장 등.
  • 클라이언트(Client) — 호스트 안에 내장되어 있으며, 서버 하나하고만 MCP 대화를 나눕니다. 우리가 신경 쓸 필요는 없습니다.
  • 서버(Server) — 실제로 "능력"을 제공합니다. 우리가 Python으로 만들려는 것이 바로 이 서버입니다.

MCP 서버가 제공하는 세 가지 능력

MCP 서버는 AI에게 세 종류의 "능력"을 노출할 수 있습니다. 지금은 이름만 눈에 익혀 두면 됩니다. 상세 설명과 실제 코드는 04장에서 다룹니다.

TOOLS

도구 (Tools)

AI가 호출할 수 있는 "행동". 함수 호출과 같습니다. 예: 지금 3호선 열차 운행 상태 조회. 가장 자주 쓰는 개념입니다.

RESOURCES

자원 (Resources)

AI가 읽어올 수 있는 "데이터 덩어리". 예: 역 목록 CSV, 운영 매뉴얼 PDF 등. 읽기 전용.

PROMPTS

프롬프트 (Prompts)

자주 쓰는 대화 템플릿을 미리 정의해서 AI에게 노출. 예: "장애 분석 리포트 양식".

철도 엔지니어라면 이런 걸 만들 수 있어요

이 가이드의 독자가 많을 것으로 예상되는 철도 분야를 예로 들어 봅니다. 모두 MCP 서버 하나로 구현 가능하며, 이 가이드의 08장에서 비슷한 예시를 직접 만들어 봅니다.

  1. 역/노선 정보 조회 서버 — "강남역의 위도·경도와 인접 역을 알려줘"를 Claude가 물어오면 사내 DB에서 조회해 반환.
  2. 시뮬레이션 실행 서버 — 기존에 MATLAB으로 짜 둔 "역 간 소요시간 예측" 시뮬레이션을 AI가 직접 호출하도록 노출.
  3. 유지보수 알림 서버 — 설비 로그를 읽어 "최근 24시간 이상 징후" 요약을 AI가 받아갈 수 있게 함.

직접 만들면 무엇이 좋은가요?

  • 사내 데이터를 외부로 내보내지 않고 AI 도구의 지능을 활용할 수 있습니다(로컬 실행 가능).
  • 한 번 만들어 두면 Claude Desktop·Cursor·다른 MCP 호환 AI에서 모두 쓸 수 있습니다.
  • Python 함수 하나 수준의 간단한 작업도 그대로 "AI가 호출할 수 있는 능력"으로 변환됩니다.
  • 나중에 다른 팀과 서버만 공유하면 그 팀도 즉시 활용할 수 있습니다.
핵심 요약

MCP는 "AI를 위한 USB-C". 우리가 Python으로 MCP 서버를 만들면, Claude를 비롯한 여러 AI 앱이 그 기능을 동일한 방식으로 호출할 수 있게 됩니다.
