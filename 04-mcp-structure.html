MCP 서버 구조 및 구성 요소

MCP 서버가 AI에게 제공하는 세 종류의 능력과, 서버와 AI 앱이 실제로 주고받는 통신 방식을 알아봅니다. 마지막에는 가장 작은 서버를 Claude Desktop에 연결해 봅니다.

예상 학습 시간 · 20분 02·03장 이수 권장

MCP 서버가 하는 일

MCP 서버는 "AI가 호출할 수 있는 능력 목록"을 갖고 있는 작은 프로그램입니다. Python 파일 하나가 서버가 될 수도 있고, 여러 파일로 쪼갤 수도 있습니다. 서버가 가진 능력은 크게 세 종류로 나뉩니다.

세 가지 구성 요소

TOOLS

Tools — 행동

AI가 실제로 호출해서 일을 시키는 함수. 계산·조회·API 호출 등. 가장 많이 쓰는 개념이며, 이 가이드의 초점입니다.

RESOURCES

Resources — 데이터

AI가 읽어가는 정적 자료. 파일·문서·DB 레코드 등. "참고 문서"를 AI에게 노출할 때 씁니다.

PROMPTS

Prompts — 템플릿

자주 쓰는 대화 템플릿을 미리 만들어 노출. 사용자가 호스트에서 "슬래시 명령"처럼 불러 쓸 수 있습니다.

초보자에게 가장 중요한 단 한 가지

실무에서 가장 많이 쓰는 건 압도적으로 Tools입니다. Resources와 Prompts는 이번 가이드에서는 존재만 알면 충분합니다. 이후 05·06·07·08장은 Tools 이야기만 합니다.

Tool은 결국 Python 함수 하나

FastMCP에서 Tool은 함수 위에 @mcp.tool 한 줄을 붙인 것과 같습니다. 별도 클래스나 복잡한 등록 과정이 필요 없습니다.

python
@mcp.tool
def hello(name: str) -> str:
    """이름을 받아 인사말을 반환합니다."""
    return f"안녕하세요, {name}님!"

이게 전부입니다. FastMCP는 이 함수 하나를 보고 다음을 자동으로 만들어 줍니다.

  • Tool 이름 (hello)
  • AI에게 보여줄 설명 (docstring)
  • 입력 스키마 (name: str)
  • 출력 스키마 (-> str)

자동 매핑의 상세 규칙은 다음 장 05장의 핵심 주제입니다.

서버와 호스트는 어떻게 대화할까?

MCP 서버와 호스트(Claude Desktop 등)는 "transport"라 부르는 통신 채널로 메시지를 주고받습니다. 주요 두 가지만 기억하면 됩니다.

Transport언제 쓰나장·단점
stdio
기본값		 내 컴퓨터에서 서버를 실행하고, 같은 컴퓨터의 Claude Desktop 같은 앱이 사용할 때 설정이 간단하고 속도가 빠름. 원격 접속은 불가능. 이 가이드는 이 방식만 사용.
Streamable HTTP
원격		 서버를 별도 호스트에 올리고 네트워크로 여러 사용자가 붙을 때 URL로 공유 가능. 인증·네트워크 설정 필요. 09장에서 소개만.
transport는 코드에서 어디서 결정되나?

mcp.run()이 기본적으로 stdio를 씁니다. 원격용으로 바꾸려면 mcp.run(transport="streamable-http", port=8000)처럼 옵션을 넘기면 됩니다. 지금은 기본값만 쓰면 됩니다.

FastMCP 내부에서 무슨 일이 일어날까?

이 절은 "호기심 차원"이며, 건너뛰어도 됩니다. 한 장의 그림이면 충분합니다.

여러분은 왼쪽 "Python 함수"만 작성하면 됩니다. 가운데 FastMCP가 나머지 복잡한 일을 전부 대신 해 줍니다.

가장 작은 MCP 서버

02장에서 이미 만들어 본 서버를 다시 한 번 천천히 살펴봅니다. 줄마다 무엇을 하는지 이해해 두면 다음 장 이후가 수월합니다.

python · hello.py
from fastmcp import FastMCP          # 1) 라이브러리 가져오기

mcp = FastMCP("hello-server")        # 2) 서버 인스턴스. 이름은 자유.

@mcp.tool                            # 3) 이 데코레이터가 함수를 Tool로 등록
def hello(name: str) -> str:
    """이름을 받아 인사말을 반환합니다."""   # 4) AI가 보는 설명
    return f"안녕하세요, {name}님!"

if __name__ == "__main__":           # 5) 직접 실행될 때만
    mcp.run()                         #    서버 시작 (기본 stdio)

Claude Desktop에 연결해 보기

서버를 "Claude"에 붙이려면, Claude Desktop의 설정 파일에 "이 파일을 이렇게 실행해 달라"고 적어 주면 됩니다.

방법 1 — 자동 등록(추천)

FastMCP에는 Claude Desktop 설정을 자동으로 수정해 주는 편리한 명령이 있습니다.

bash
fastmcp install claude-desktop hello.py

Claude Desktop을 완전히 종료했다가 다시 실행하면 "hello-server"가 도구 목록에 나타납니다.

방법 2 — 수동 등록

자동 등록이 안 되거나 원리를 확인하고 싶다면, Claude Desktop 설정 파일을 직접 편집합니다.

Windows: %APPDATA%\Claude\claude_desktop_config.json

macOS: ~/Library/Application Support/Claude/claude_desktop_config.json

json · claude_desktop_config.json
{
  "mcpServers": {
    "hello-server": {
      "command": "python",
      "args": ["C:\\mcp-first\\hello.py"]
    }
  }
}
경로 주의

Windows에서는 경로 구분자 \가 JSON에서 특수문자라서 두 번 겹쳐(\\) 적어야 합니다. 또는 /를 써도 됩니다. 가상환경 안의 Python을 정확히 지정하려면 command를 C:\\mcp-first\\.venv\\Scripts\\python.exe처럼 절대 경로로 주는 것이 안전합니다.

연결이 되면?

Claude Desktop의 대화창 아래 도구 아이콘에 "hello-server"가 뜨고, "이름이 '미나'인 사람에게 인사해줘"라고 말하면 Claude가 우리가 만든 hello Tool을 호출합니다.
