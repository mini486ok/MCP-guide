철도 정보 MCP 서버 만들기

이 장에서는 지금까지 배운 내용을 모두 합쳐 "철도 정보 MCP 서버"를 처음부터 끝까지 만들어 봅니다. 4개 Tool과 2개 데이터 모델이 포함된 작지만 완결된 예제입니다.

예상 학습 시간 · 30~40분 실습 05·06장 선행

만들 것

가상의 사내 철도 정보 조회 서버입니다. 실제 API 연결 없이 하드코딩된 샘플 데이터(서울 지하철 1·2호선 일부)만으로 동작합니다.

TOOL 1

list_stations

관리 중인 모든 역을 목록으로 반환. 호선 번호로 필터링 가능.

TOOL 2

get_station_info

역 이름을 받아 호선·환승 여부·좌표 등 상세 정보 반환.

TOOL 3

distance_between

두 역 이름을 받아 하버사인 공식으로 직선 거리(km) 계산.

TOOL 4

find_trains

출발·도착역에 맞는 가상의 열차 시간표를 조회. 정렬 기준 선택 가능.

Step 2. 데이터 모델 정의

응답으로 돌려줄 데이터 구조를 Pydantic BaseModel로 먼저 정의합니다. 이렇게 하면 outputSchema가 정확하게 잡히고, 오타·잘못된 타입이 자동으로 걸러집니다.

python
from __future__ import annotations

from math import asin, cos, radians, sin, sqrt
from typing import Literal

from pydantic import BaseModel, Field

from fastmcp import FastMCP


class Station(BaseModel):
    """역 정보."""
    name: str = Field(description="역 이름")
    line: int = Field(description="호선 번호")
    is_transfer: bool = Field(description="환승역 여부")
    lat: float = Field(description="위도")
    lng: float = Field(description="경도")


class Train(BaseModel):
    """열차 운행 정보."""
    train_no: str = Field(description="열차 번호")
    departure: str = Field(description="출발역 이름")
    arrival: str = Field(description="도착역 이름")
    depart_time: str = Field(description="출발 시각(HH:MM)")
    arrive_time: str = Field(description="도착 시각(HH:MM)")
    duration_min: int = Field(description="총 소요 시간(분)")


class DistanceResult(BaseModel):
    """두 역 사이 거리 계산 결과."""
    departure: str
    arrival: str
    distance_km: float = Field(description="직선 거리(킬로미터)")

각 필드의 description이 그대로 AI가 보는 스키마에 들어갑니다. 필드 의미를 한 줄씩 써 두는 게 미래의 나를 돕습니다.

Step 4. 거리 계산 "코어 함수" + 역 이름 정규화

07장에서 배운 대로, 재사용 가능한 순수 함수로 먼저 분리합니다. 이 함수들은 MCP와 무관하게 독립적으로 테스트 가능합니다.

python
def _haversine_km(lat1: float, lng1: float, lat2: float, lng2: float) -> float:
    """위·경도를 받아 지표 위 직선 거리(km)를 돌려주는 순수 함수."""
    earth_radius_km = 6371.0
    dlat = radians(lat2 - lat1)
    dlng = radians(lng2 - lng1)
    a = sin(dlat / 2) ** 2 + cos(radians(lat1)) * cos(radians(lat2)) * sin(dlng / 2) ** 2
    return 2 * earth_radius_km * asin(sqrt(a))


def _normalize_station(name: str) -> str:
    """'강남' / '강남역' 같은 입력 변형을 STATIONS 키로 정규화."""
    if name in STATIONS:
        return name
    if name.endswith("역") and name[:-1] in STATIONS:
        return name[:-1]
    if (name + "역") in STATIONS:
        return name + "역"
    available = ", ".join(STATIONS.keys())
    raise ValueError(
        f"'{name}' 역은 이 서비스가 관리하는 역 목록에 없습니다. "
        f"가능한 역: {available}"
    )

함수 이름 앞의 _는 "외부에 공개하지 않는 내부 함수"라는 관례입니다. MCP Tool은 _로 시작하지 않는 공개 함수들로 구성합니다.

왜 정규화 헬퍼가 필요할까

LLM(Claude)은 같은 역을 사람마다 다르게 부릅니다. "강남에서 잠실까지"라고 묻는 사람도 있고 "강남역에서 잠실역까지"라고 묻는 사람도 있죠. 데이터의 키와 표기가 한 글자라도 다르면 Tool이 빈 결과를 돌려주거나 오류를 내고, 그러면 LLM은 "정보가 없습니다"로 답을 마무리해 버립니다. 이 헬퍼 한 함수로 두 표기를 모두 받아주고, 매칭 실패 시 가능한 역 목록을 오류 메시지에 함께 노출해서 LLM이 다음 시도에서 자가복구할 수 있게 만듭니다(07장의 "오류 메시지를 LLM이 읽는다" 원칙).

Step 5. 서버 인스턴스 + 4개의 Tool

Tool 1: list_stations

python
mcp = FastMCP("railway-info")


@mcp.tool
def list_stations(line: int | None = None) -> list[Station]:
    """등록된 역 목록을 반환합니다.

    Args:
        line: 지정하면 해당 호선의 역만 필터링. 생략하면 전체 반환.

    Returns:
        Station 객체의 리스트.
    """
    stations = list(STATIONS.values())
    if line is not None:
        stations = [s for s in stations if s.line == line]
    return stations

line 인자에 기본값 None이 있으므로 스키마에서 선택 인자가 됩니다. 호출 예: list_stations() 또는 list_stations(line=2).

Tool 2: get_station_info

python
@mcp.tool
def get_station_info(name: str) -> Station:
    """역 이름으로 역 상세 정보를 조회합니다.

    역 이름은 '강남' / '강남역' 둘 다 받습니다('역' 접미사 유무 무관).
    어떤 역이 등록되어 있는지 확신이 없다면 먼저 list_stations()를 호출하세요.

    Args:
        name: 조회할 역의 한글 이름(예: "강남" 또는 "강남역").

    Returns:
        일치하는 Station 객체.

    Raises:
        ValueError: 등록되지 않은 역 이름을 전달한 경우. 오류 메시지에
            가능한 역 목록이 함께 포함됩니다.
    """
    key = _normalize_station(name)
    return STATIONS[key]

07장의 팁 4에 따라 오류 메시지를 한국어 문장으로 썼고, 가능한 역 목록을 함께 돌려줍니다. AI가 그 메시지를 받아 "그럼 강남, 잠실 중에 어디를 보시겠어요?"처럼 자연스럽게 이어갈 수 있습니다.

Tool 3: distance_between

python
@mcp.tool
def distance_between(station_a: str, station_b: str) -> DistanceResult:
    """두 역 사이의 직선 거리(km)를 계산합니다.

    역 이름은 '강남' / '강남역' 둘 다 받습니다('역' 접미사 유무 무관).

    Args:
        station_a: 첫 번째 역 이름.
        station_b: 두 번째 역 이름.

    Returns:
        거리 정보(DistanceResult). departure/arrival 필드에는 정규화된
        역 이름이 들어갑니다.

    Raises:
        ValueError: 둘 중 하나라도 등록되지 않은 역인 경우.
    """
    key_a = _normalize_station(station_a)
    key_b = _normalize_station(station_b)
    a = STATIONS[key_a]
    b = STATIONS[key_b]
    km = _haversine_km(a.lat, a.lng, b.lat, b.lng)
    return DistanceResult(
        departure=key_a,
        arrival=key_b,
        distance_km=round(km, 3),
    )

Tool 4: find_trains

python
@mcp.tool
def find_trains(
    departure: str,
    arrival: str,
    sort_by: Literal["depart_time", "duration_min"] = "depart_time",
) -> list[Train]:
    """출발역과 도착역이 일치하는 열차 시간표를 조회합니다.

    역 이름은 '강남' / '강남역' 둘 다 받습니다('역' 접미사 유무 무관).
    "가장 빠른 열차"를 묻는 질문이라면 sort_by="duration_min"으로 호출하세요
    (기본값 'depart_time'은 가장 일찍 출발하는 순입니다).

    Args:
        departure: 출발역 이름.
        arrival: 도착역 이름.
        sort_by: 결과 정렬 기준. 'depart_time'(가장 일찍 출발 순) 또는
            'duration_min'(가장 빠르게 도착 순).

    Returns:
        조건에 맞는 Train 객체 리스트. 일치 항목이 없으면 빈 리스트.

    Raises:
        ValueError: 출발/도착역 중 하나라도 등록되지 않은 역인 경우.
    """
    dep = _normalize_station(departure)
    arr = _normalize_station(arrival)
    results = [t for t in TRAINS if t.departure == dep and t.arrival == arr]
    results.sort(key=lambda t: getattr(t, sort_by))
    return results

Literal["depart_time", "duration_min"]로 쓴 덕분에 AI는 저 두 값 외에는 시도하지 않습니다. 잘못된 정렬 기준이 들어올 가능성을 원천 차단했습니다. 또 docstring 안에 "'가장 빠른'은 duration_min"이라고 적어두면, "아침에 가장 빠른 열차" 같은 질문이 와도 AI가 올바른 정렬 기준을 고를 가능성이 높아집니다.

실습 시나리오 3가지
  1. 정상 호출: list_stations를 아무 인자 없이 호출. 10개 역이 나오는지 확인.
  2. 오류 확인: get_station_info에 "도쿄역"을 넣어 호출. ValueError 메시지가 그대로 보이는지 확인(= AI도 저 메시지를 읽는다는 뜻).
  3. 열거형: find_trains의 sort_by에 "random"을 넣어 Run. Inspector가 스키마에 맞지 않는 값이라고 거부하는지 확인.
